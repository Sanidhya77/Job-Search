## Note on use

These reports describe the project as it was *planned*, not as it is *implemented*. The implementation may diverge from the plan as decisions are revisited during coding. When that happens, the change and its reasoning will be recorded in `journal/dev_log.md` rather than rewritten back into these planning documents, so the planning history stays intact.
